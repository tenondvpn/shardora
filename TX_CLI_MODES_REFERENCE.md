# tx_cli 模式参考指南

## 所有可用模式

### Mode 0: 标准压力测试

**用途**: 使用现有账户进行持续压力测试

**命令**:
```bash
./txcli 0 <net_id> <pool_id> <ip> <port> [delay_us] [multi_pool]
```

**示例**:
```bash
./txcli 0 3 0 127.0.0.1 13001
```

**特点**:
- 使用 `init_accounts` 文件中的账户
- 随机选择 to 地址（从 g_addrs）
- 持续运行直到手动停止
- 实时显示 TPS

---

### Mode 1: 部署智能合约

**用途**: 编译并部署 Solidity 合约

**命令**:
```bash
./txcli 1 <shard> <pool> <ip> <port>
```

**示例**:
```bash
./txcli 1 3 0 127.0.0.1 13001
```

**特点**:
- 编译 `../python/ex.sol`
- 部署到区块链
- 保存合约地址到 `ex_contract_addr.txt`

---

### Mode 2: 合约功能测试

**用途**: 测试已部署合约的所有功能

**命令**:
```bash
./txcli 2 <shard> <pool> <ip> <port>
```

**示例**:
```bash
./txcli 2 3 0 127.0.0.1 13001
```

**特点**:
- 读取 `ex_contract_addr.txt` 中的合约地址
- 测试所有合约方法
- 显示测试结果

**前置条件**: 必须先运行 Mode 1

---

### Mode 3: 合约压力测试

**用途**: 对合约进行高并发调用测试

**命令**:
```bash
./txcli 3 <shard> <pool> <ip> <port> [threads] [items_per_thread]
```

**示例**:
```bash
# 使用 4 个线程，每个线程 100 个项目
./txcli 3 3 0 127.0.0.1 13001 4 100

# 使用 16 个线程，每个线程 500 个项目
./txcli 3 3 0 127.0.0.1 13001 16 500
```

**特点**:
- 预创建测试数据
- 多线程并发调用合约
- 速率限制（100 TPS）
- 实时显示统计信息

**前置条件**: 必须先运行 Mode 1

---

### Mode 4: 10,000 账户压力测试 ⭐ NEW

**用途**: 大规模账户随机转账压力测试

**命令**:
```bash
./txcli 4 <shard> <pool> <ip> <port> [threads]
```

**示例**:
```bash
# 使用默认 16 个线程
./txcli 4 3 0 127.0.0.1 13001

# 使用 32 个线程
./txcli 4 3 0 127.0.0.1 13001 32
```

**特点**:
- 生成 10,000 个随机账户
- 创建账户并转入初始余额
- 验证账户创建成功
- 10,000 个账户之间随机转账
- 实时 TPS 监控
- 自动 Nonce 管理

**执行阶段**:
1. Phase 1: 生成 10,000 个账户（~10 秒）
2. Phase 2: 创建账户（~45 秒）
3. Phase 3: 等待确认（30 秒）
4. Phase 4: 验证账户（~30 秒）
5. Phase 5: 压力测试（持续运行）

## 故障排除

### 问题 1: 编译错误

**错误**: `undefined reference to CreateTransactionWithAttr`

**解决**: 确保所有依赖文件都已编译

```bash
make clean
make -j8
```

---

### 问题 2: 连接失败

**错误**: `Failed to send tcp client`

**解决**: 
1. 检查 Shardora 节点是否运行
2. 验证 IP 和端口
3. 检查防火墙设置

---

### 问题 3: Nonce 错误

**错误**: 大量交易失败

**解决**:
1. 增加 nonce 更新频率
2. 减少并发线程数
3. 增加交易延迟

---

### 问题 4: 内存不足

**错误**: `std::bad_alloc`

**解决**:
1. 减少账户数量（Mode 4）
2. 增加系统内存
3. 优化数据结构

## 最佳实践

```bash
# 确保 Shardora 节点运行
ps aux | grep shardora

# 检查账户余额
```
